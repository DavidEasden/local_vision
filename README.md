# Local Vision MCP

A native visual analysis tool based on MCP (Model Context Protocol) that uses LM Studio's vision model to analyze images.

## Functional features

- 🔍 **Local Image Analysis**: Supports analyzing images in local PNG, JPG, JPEG, WEBP formats
- 🤖 **Visual Model Integration**: Integrate LM Studio's vision models for image understanding
- 🛡️ **Security Restrictions**: Only allow access to locally existing image files to prevent path traversal attacks
- ⚡ **Fast Response**: Asynchronous processing with support for custom analysis prompts

## System Requirements

- Python 3.8+
- The LM Studio(or other local server) service is running
- Supported image formats: PNG, JPG, JPEG, WEBP

## Installation configuration

### 1. Environmental preparation(Unnecessary)

Make sure Conda is installed and create an 'mlx' environment:

```bash
conda create -n mlx python=3.10
conda activate mlx
```

### 2. Install dependencies

To install the required dependencies in the project directory:

```bash
pip install mcp httpx
```

### 3. Configure LM Studio

1. Launch LM Studio
2. Download and load a vision model (e.g. 'qwen3.5:2b-bf16')
3. Make sure the LM Studio API service is running at 'http://localhost:11434'

### 4. Environment Variable Configuration (Optional)

The following environment variables can be set up:

```bash
# Set the LM Studio service address (default: http://localhost:11434)
export LM_STUDIO_URL="http://localhost:11434"

# Set the visual model name (default: qwen3.5:2b-bf16)
export VISION_MODEL="qwen3.5:2b-bf16"

# Set the Conda environment name (default: mlx)
export MCP_CONDA_ENV="mlx"
```

## How to use

### Runs as an MCP server

1. **Direct Run**:

```bash
python main.py
```

2. **Configure in Opencode**:

Add in your ```opencode.json```:

```json
{
 "mcp": {
    "local_vision": {
      "type": "local",
      "command": ["python", "/Users/david/MyProjects/MLX/local_vision_mcp/main.py"],
      "environment": {
        "LM_STUDIO_URL": "http://localhost:11434",
        "VISION_MODEL": "qwen3.5:2b-bf16"
      }
    }
}
```

### Available Tools

#### 'analyze_image' - Analyze local images

**Parameters**:
- 'image_path' (required): The local absolute path to the image file
- 'Prompt' (optional): For specific questions asked about the image, the default prompt will ask for a description of the image's content

Example Calls:

```json
{
  "name": "analyze_image",
  "arguments": {
    "image_path": "/Users/username/Pictures/photo.png",
    "prompt": "What objects are in the image? What is their positional relationship? "
  }
}
```

**Default prompt**:
```
Please answer strictly based on the information visible in the picture, do not guess. First, outline the image, then extract the visible text, explain the key elements and positional relationships, and finally answer my question directly. Where you can't see clearly, please clearly write "uncertain". My question: Please describe this image
```

## How it works

1. **Environment Check**: Automatically detects and switches to the specified Conda environment
2. **File Verification**: Check that the image file exists and the format is supported
3. **Image Encoding**: Convert the image to base64 format
4. **Model Calls**: Call the vision model through the LM Studio API
5. **Result Interpretation**: Extracts and returns the text description generated by the model

## Security features

- ✅ Only allow access to locally existing files
- ✅ Restrict file extensions (image formats only)
- ✅ Prevent path traversal attacks
- ✅ Input parameter validation

## Troubleshooting

### FAQs

1. **"Python not found for Conda environment mlx"**
   - Make sure the 'mlx' environment is created: 'conda create -n mlx python=3.10'
   - Or set the 'MCP_CONDA_ENV' environment variable to specify a different environment name

2. **Failed to call the visual model**
   - Check if LM Studio is running: 'curl http://localhost:11434/v1/models'
   - Confirm that the model is loaded and supports vision capabilities
   - Check your network connection and firewall settings

3. **Unsupported File Formats**
   - Make sure the image format is: PNG, JPG, JPEG, WEBP
   - Check that the file extension is correct

4. **'File does not exist'**
   - Verify that the image path is correct and the file exists
   - Use absolute paths instead of relative paths
